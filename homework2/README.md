# Movie Theater Booking Application

A RESTful Movie Theater Booking Application built with Python and Django. The application allows users to view movies, book seats, and check their booking history through both an attractive web interface and a complete REST API.

## Features

- **Movie Listings**: Browse available movies with details
- **Seat Booking**: Book seats for movies with real-time availability
- **Booking History**: View your past bookings
- **RESTful API**: Full API endpoints for all operations
- **Admin Panel**: Manage movies, seats, and bookings
- **Bootstrap UI**: Responsive and attractive user interface

## Tech Stack

- Python 3.x
- Django 6.0.3
- Django REST Framework 3.16.1
- SQLite (development) / PostgreSQL (production)
- Bootstrap 5.3

## Setup Instructions

### Local Development

1. **Clone the repository**
   ```bash
   cd homework2
   ```

2. **Create Virtual Environment**
   ```bash
   python3 -m venv myenv
   source myenv/bin/activate  # On Windows: myenv\Scripts\activate
   ```

3. **Install Dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Apply Migrations**
   ```bash
   python manage.py migrate
   ```

5. **Populate Database with Test Data**
   ```bash
   python manage.py populate_db
   ```

6. **Create Superuser (Optional)**
   ```bash
   python manage.py createsuperuser
   ```

7. **Run Development Server**
   ```bash
   python manage.py runserver 0.0.0.0:3000
   ```

8. **Access the Application**
   - Web Interface: http://localhost:3000/
   - Admin Panel: http://localhost:3000/admin/
   - API: http://localhost:3000/api/

## API Endpoints

### Movies
- `GET /api/movies/` - List all movies
- `POST /api/movies/` - Create a new movie
- `GET /api/movies/{id}/` - Get movie details
- `PUT /api/movies/{id}/` - Update a movie
- `DELETE /api/movies/{id}/` - Delete a movie

### Seats
- `GET /api/seats/` - List all seats
- `GET /api/seats/available/` - List available seats
- `POST /api/seats/` - Create a new seat

### Bookings
- `GET /api/bookings/` - List user's bookings
- `GET /api/bookings/history/` - Get booking history
- `POST /api/bookings/` - Create a new booking

## Running Tests

```bash
python manage.py test
```

This will run all 20 unit tests covering:
- Model tests (Movie, Seat, Booking)
- View tests (Movie list, booking creation)
- API tests (Serialization and CRUD operations)

## Deployment to Render

### Steps

1. **Create a Render Account**
   - Go to https://render.com/
   - Sign up and connect your GitHub repository

2. **Deploy Using render.yaml**
   - Render will automatically detect `render.yaml`
   - Configuration includes:
     - Web service for Django app
     - PostgreSQL database
     - Automatic migrations on deploy

3. **Environment Variables**
   - `DEBUG`: Set to `false` in production
   - `SECRET_KEY`: Generated by Render
   - `ALLOWED_HOSTS`: Configured for Render domains
   - `DATABASE_URL`: Connected to PostgreSQL

4. **Monitor Deployment**
   - Check logs in Render dashboard
   - Application will be available at `https://<service-name>.onrender.com/`
